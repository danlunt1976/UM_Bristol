Introduction¶

The aim of this worksheet is to give you some initial experience using Python, to enable you to gain confidence and train your Python 'muscles'!

Why Python?¶

Over the years, I have used many different languages to visualise and/or analyse climate model outputs, including IDL, ferret, VCS, Grads, ncl, ncview, xconv, Panoply, fortran, and seveal others! They come in and out of fashion, but they all have strengths and weaknesses. Some are good for just viewing data very quickly (e.g. xconv), others are very fast for analysis (e.g. fortran), and others make very nice publication-quality plots (e.g. IDL).

In this handbook we will use Python. Currently, Python is widely used in climate science as it has a huge online community, has many useful built-in functions for handling spatial data, and is great for publication-quality plots. It is also widely used outside of climate science, and so coding in Python is a useful skill to have. However, all programming languages fundamentally behave the same way with the basic functionalities, such as variables, functions, loops, and so on. As such, by learning soome Python, you are learning valuable and transferable skills, which will be applicable to many other languages, including, for example R or Matlab.

Making a new Jupyter notebook¶

Launch Jupyter notebook. It will open a homepage in your browser. From there, you can access your local and/or cloud file structure. First navigate to the directory that you wish to use for this handbook - this will be your 'working directory'. Then start a new Jupyter notebook by clicking File > New Notebook > Python 3 (ipykernel). Give your notebook a recognisable name, and make sure that you can see it in the working directory.

Within your new notebook, you can add 'cells' which will either contain code, or text (written in the 'markdown' language https://www.markdownguide.org/basic-syntax/ ). To run a cell, highlight that cell, and then click on the 'play'/'Run' button. When a cell is running, it is preceded by a '*'; when a cell has completed successfully, it is preceded by a number. You can also export your file to html or other formats; this handbook itself is written as a Jupyter notebook and exported to html.

Hints and Tips for programming¶

Here are some pointers which may come in useful.

  • First of all, and most important, the internet and Large Language Models (LLMs, such as ChatGPT) are your friend!! Any problem you have with Python, you can be sure that thousands of people will previously have had the same problem, and LLMs are trained on information from online forums, as well as coding sites such as github. So make use of the massive online community (e.g. Stackoverflow), and LLMs, and just search/ask about any commands you don't understand, or tasks you are struggling to complete.

  • Make sure that you keep your Notebook organised - ensure that yur code will always work correctly if cells are run in order from the top to the bottom.

  • You should keep copies of working versions of your work (i.e. your .ipynb file) as you go along. If you make a mistake, it can sometimes be difficult to back-track, meaning that you may have to start from scratch! Do not overwrite old versions, but keep copies with date and timestamps in the filename. This is called 'version control'. As you become experienced, eventualyl you can automate this proces using vesion control systems such as github, but for this exercise you can do your own version control manually as described above.

  • Comment your code as if you were writing the code and the comment for someone who has no idea what you are doing. Keep your coding style consistent as this improves readability. These are generic programming guidelines to get used to and adopt throughout your career and are not limited to Python or this exercise. This ultimately will help you when you want to understand why something is not working as it should and for other people reading your code. This is also helpful to understand your code long after you wrote it.

  • Name your variables and functions in a sensible manner which reflects their real scientific meaning. Try and keep names short and to the point. For example a variable name of the form Difference_betwen_two_simulations may not be as readable as temp_diff especially if you put a comment explaining what the variable is when you use it the first time.

  • If your code doesn't work, or behaves differently from expected, then you can 'debug' it. This often involves putting lots of print() commands in the code, so you can track exactly what is happening to the variables as the code runs (see examples below).

  • If you try all the above, and your code still does not work, then do not despair! Do come and ask me if you get completely stuck, and I'll be very happy to help!

Coding Concepts¶

Variables¶

The first concept is that of a 'variable'. Variables in programming are essentially names given to a particular part of the computer 'memory'. For example, we could define a variable called dog and set it equal to 1. We can perform arithmetic operations (e.g. plus,minus,times,divide = +,-,* etc.) on variables, and they will change. Note that when faced with an '=' sign, the computer will calculate the value of whatever is to the right of the equal sign, and assign that value to the variable to the left of the equals sign. So, to define a variable called horse and set it equal to 5, and then double it, we would do:

In [1]:
print('starting!')
horse=5
print(horse)
horse=horse*2
print(horse)
print('finished!')

10
finished!

Note that in all the code examples given in this worksheet, the code itself is in grey boxes, and the output from the code is shown directly below the grey box. The 'starting' and 'finishing' print statements are just so that you cn see clearly where the output of the coe starts and finishes - you don't necessarily need to include these in yor own code.

Loops¶

A key programming concept is that of a 'loop'. Loops allow the computer to carry out the same instructions multiple times, which is basically exactly what computers are designed for!

For the next script, create a loop as follows (any line that starts with a # is a comment and is ignored by Python; it is good practice to include comments in your script, to aid understanding). Be careful not to confuse i with 1. In the code below, i is a variable, just as horse was above.

Note that Python is very sensitive to the correct use of 'tabs' in the code - the contents of a loop must be consistently idented with a tab.

In [2]:
print('starting!')
# the following line means "create a variable called i. Set i 
# initially equal to 0, and then carry out the commands that are indened by a tab (i.e. the loop).  
# Then come back to the top of the indented text and
# increment i by one (e.g. if it was 1, then set it to 2), and repeat
# until we have done this 10 times (i.e. until i is equal to 9)."
# Or in other words, "carry out the indented commands 10 times". 
for i in range(10):
# now follow the commands within the loop
# we will just ask the computer to print 'hello' to the screen.
    print('hello')

# now finish the loop with some non-indented text
print('finished!')

Scalars, Vectors, and Arrays¶

In programming, another key concept is that of scalars, vectors, and arrays. A scalar is simply a variable with a single value (that can change). dog and i in the listings above are both scalars. One way of thinking of a scalar is a single cell of an Excel (or Google Sheets) spreadsheet.

A vector is a variable which has several different 'compartments', with each compartment containing a single value. Think of a vector like a row or column of numbers in a spreadsheet. The number of compartments in a vector is often called the 'size' of the vector. The compartments are sometimes called 'elements'. When coding, you can either perform operations on an entire vector, or on an individual element of a vector. Each element has an 'index' which identifies it.

The concept is best illustrated with some code, but first we will need to import a Python library, called 'numpy'. Numpy is a very useful library which comes with nearly all Python installs, so it should be available on your machine. So you don't need to keep typing 'numpy', we import it with a shorthand 'np'.

If the 'import' line below gives an error, then it may be that numpy is not included in your version of anaconda. If this is the case, then you can install numpy by including the line !pip install numpy in a cell and running it. You will only need to do this one time - once you have done this it will be installed forever. You will need to click on 'Restart Kernel' under the 'Kernel' tab after installing, otherwise the import command below will likely not work. Note that restarting the kernel also removes all variables from Python's internal memory.

In [4]:
import numpy as np
In [5]:
print('starting!')
# first of all, let's define a vector, called 'cat' with 5 elements (i.e. a
# vector of size 5).  The 'np.zeros' command initialises each element of
# the vector as zero: 
cat=np.zeros(5)
print(cat)
# Here we will perform an operation on all the elements of cat:
cat=cat+3
print(cat)
# Here we will perform an operation on just one element (the fifth) of cat 
#  (remember that Python starts its index counting at zero, so '4' is the 5th element!)...
# We use square brackets to access a part of a vector:
cat[4]=cat[4]*2
print(cat)
# ...or the fourth element:
cat[3]=cat[3]*5
print(cat)
# Python also has a "shortcut" for the final element of a vector, which is '-1', so
cat[-1]=cat[-1]+2
print(cat)
print('finished!')

An array takes a vector and 'expands' it into another dimension. A 1-dimensional array is identical to a vector. A 2-dimensional array (going back to the analogy above), is like an entire spreadsheet, with both rows and columns. A 3-dimensional array is like an entire spreadsheet, but with multiple 'pages' or 'tabs', each containing a spreadsheet. A 4-dimensional (or higher) array is hard to visualise in this way, but is simply an extension of the above concept. By the way, by extension, a 0-dimensional array is a scalar!

In climate science, we use 2-dimensional arrays a lot, as this can be used to represent data covering the globe, for example on a latitude-longitude grid. We also use 3-dimensional variables as this can represent data over the entire atmosphere and ocean, for example on a latitude-longitude-height grid. We also use 4-dimensional variables as this can represent data over the entire atmosphere and ocean and through time, for example on a latitude-longitude-height-time grid. In this exercise, we will mostly use 2-dimensional fields (e.g. a map of temperature over the globe), so let's introduce a 2-dimensional array:

Making Global Map Plots¶

You can imagine the final values of 'penguin' representing temperature over the globe, with temperatures of 5 $^\circ$C in the Northern Hemisphere (30 degrees North to 90 degrees North), temperatures of 4 $^\circ$C in the Southern Hemisphere (30 degrees South to 90 degrees South), and most temperatures in the tropics (30 degrees South to 30 degrees North) of 10 $^\circ$C, but with one region of longitude in the tropics (60 degrees West to 0 degrees West) of 11 $^\circ$C.

Given this interpretation, we can now try to plot this as a map of temperature over the globe!

Setting up the grid¶

Before making the plot, we need to define what the longitudes and latitudes are. We'll use vectors called lon and lat for this, and define them according to the interpretaton above. Note that we define the longituds and latitudes as the centre values, so:

In [8]:
print('starting!')
# we can define a vector directly by just putting the values required in quare brackets:
lon=[-150, -90, -30, 30, 90, 150]
lat=[60,0,-60]
print(lon)
print(lat)
print('finished!')

[60, 0, -60]

Making the plot¶

We will also use another library called matplotlib, which is included in most python installs, but if not you can install it following the instructions above by analogy.

In [9]:
import matplotlib.pyplot as plt
import cartopy.crs as ccrs

Now we are ready to make out plot:

In [10]:
# size 10 by 6 inches gives a nice aspect ratio
# 'fig' will be the variable name of our figure (we can use any name, but if you change this here you will also need 
#   to change it in the code below).
# 'plt.figure' is the matplotlib routine to set up the plot, and figsize is the width,height of the plot in inches.
fig = plt.figure(figsize=(10, 6))

# The Robinson projection is standard for IPCC.  We'll centre on 0 longitude.
# 'ax' will be the variable naem of our axes.
# 'plt.axes' id the matplotlib routine to set up the plot axes, and ccrs.Robinson is the caropy routine to make a 
#   Robinson projection
ax = plt.axes(projection=ccrs.PlateCarree(central_longitude=0))


# add dashed gridlines with a width of 1.0, in light-grey, and 50% transparency
# 'gl' will be the variable name for our gridlines on the plot
# 'ccrs.PlateCarree()' is also related to the projection; the other options define our gridlines 
gl = ax.gridlines(crs=ccrs.PlateCarree(), draw_labels=True,
                  linewidth=1, color='lightgray', alpha=0.5, linestyle='--')
# This next makes the plot gridlines look less cluttered, removing the top and right labels
# (note that there is a bug in cartopy in that right_labels is False but it still prints 60oN/S annoyingly!)
gl.top_labels = False
gl.right_labels = False

# Define the colour bar limits.  We'll go from 3 (min_level) to 12 (max_level), in a step of 1 (step_level).
#   This will give us 9 levels in total.
min_level=3
max_level=12
step_level=1
# 'np.arrange' is a numpy function to create a vector that starts at the first argument, stops at the second argument, 
#   and goes up in stes of the third argument.  Try 'print(my_levs)' to convince yourself this has worked! 
my_levs=np.arange(min_level, max_level+step_level, step_level)

# Make the contour plot
#  'plt.contourf' is a matplotlib function that creates a contour plot.
#  The first argument is the x axis (lon), the second is the y axis (lat), and thr third is the variable 
#  to be plotted (penguin).  'levels' is the ontour leves tha we set up above.  The 'transform' puts the 
#   data into the correct projection. 
contour=plt.contourf(lon, lat, penguin, transform=ccrs.PlateCarree(central_longitude=0),levels=my_levs)

# Make the colour legend (add_axes positions the colour bar, [left, bottom, width, height])
colorbar_axes = fig.add_axes([0.95, 0.2, 0.02, 0.6])
cbar=plt.colorbar(contour,cax=colorbar_axes,label='Temperature [degrees C]')

# add a modern continental outline:
ax.coastlines()

# output to the screen:
plt.show()
